介绍：
本系统为成绩录入系统。基本实现了不同格式与不同格式的输出，并且兼具了一定的可扩展性。
默认测试路径为Source/Test.csv或者Source/Test.json

架构思路：
本系统采用简化版的MVC框架思路。
将外部输入、输出作为view层；
因为简单所以无需controller层进行分发，controller层作为一般mvc中的第二层model用于实现复杂的数据计算；
model层作为底层直接存放数据。

在view层输入时，针对不同输入格式，采取策略模式的方式来进行不同算法的切换，以达到最终输入数据的目的，并且拥有良好的可扩展性。这个思路同样使用在了输出格式的切换中。
对于策略对象的使用，借鉴了RPC中的客户端调用服务端方法思路，使用一个类似工厂模式的思路。首先用工厂类将所有的策略模式类生成对象，然后在工厂中使用map，以名字（string类型）作为key值，对应不同的策略对象。然后调用时只需要传递名字就可以直接获得想要的策略并执行。
同时由于Controller和Factory的频繁使用特性，对他们实现了单例模式，方便进行方法的调用。
